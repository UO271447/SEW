Conceptos Basicos

Según la empresa, Ember consta de cinco conceptos clave:

  • Routes(URL)
    • En Ember, el estado de una aplicación está representado por una URL
  • Modelos
    • Cada ruta tiene un modelo asociado
  • Plantillas
    • Las plantillas se utilizan para construir el HTML de la aplicación
  • Componentes
    • Un componente es una etiqueta HTML personalizada
  • Servicios
    • Los servicios son solo objetos únicos para almacenar datos de larga duración.
